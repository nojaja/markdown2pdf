# markdown2pdf CLI 設計書（v1.1.0）

## 1. 概要
実行環境が社内Proxyなどで外部ネットワークへ接続できない場合でも、Mermaid記法を含むMarkdownをPDF変換できるようにする。

本バージョンでは、変換時に必要なレンダリングモジュールをパッケージへ同梱し、実行時の外部アクセスを不要化する。

対象Issue:
- https://github.com/nojaja/markdown2pdf/issues/1

## 2. 背景と課題
- 現在のHTMLテンプレートはCDN経由でMermaidを読み込んでいる。
- オフライン環境ではCDNアクセスに失敗し、Mermaid図の描画ができない。
- その結果、変換成功率が環境依存となり、CLIの可搬性を損なう。

## 3. ユースケース
1. 社内閉域網で運用されるCIサーバー上で、ネットワーク接続なしでMarkdownをPDFへ変換する。
2. 開発端末がProxy制限下にあり、外部CDNへ接続できない状態でMermaid図を含む資料を出力する。
3. オフライン環境の自動バッチで日次帳票を安定生成する。

## 4. 機能要件

### 4.1 Mermaidライブラリの同梱
- 依存関係に `mermaid` を追加し、インストール時にローカルへ配置されること。

### 4.2 CDNスクリプト参照の廃止
- HTMLテンプレートから以下のCDN参照を削除すること。

  `<script src="https://cdn.jsdelivr.net/npm/mermaid@10/dist/mermaid.min.js"></script>`

### 4.3 ローカルリソース読込への置換
- 変換時にMermaidをローカル依存から読込む実装へ置き換えること。
- 変換処理は外部ネットワーク通信を前提にしないこと。

### 4.4 Mermaid初期化の維持
- Mermaid初期化処理（startOnLoad相当）は従来通り有効であること。
- 既存サンプル（ガントチャートを含む）で描画結果を維持すること。

## 5. 非機能要件

### 5.1 ネットワーク非依存
- 変換処理中に外部URLへのアクセスが不要であること。

### 5.2 互換性
- CLIの既存コマンド体系とオプション互換性を維持する。
- 既存のMarkdown変換パイプライン（Markdown->HTML->PDF）を維持する。

### 5.3 再現性
- ネットワーク状態に依存せず、同一入力で同一レイアウト結果が得られること。

## 6. インターフェース定義

### 6.1 CLIインターフェース
- 追加・変更オプションなし（既存CLI仕様を維持）。

### 6.2 依存関係インターフェース
- package.json
  - dependencies に `mermaid` を追加。

### 6.3 内部実装インターフェース
- HTMLテンプレート構築部
  - CDN URLハードコードを廃止。
  - ローカル依存のMermaid読込方式へ変更。

## 7. 設計方針
1. 実行時の外部取得を禁止する。
2. 依存はnpmパッケージとして配布物に含める。
3. 既存の描画初期化タイミングは変更しない。
4. 既存CLI利用者の操作方法は変えない。

## 8. 互換性と移行計画
- 利用者側のCLI操作変更は不要。
- 導入時は通常の依存インストールを実施するのみ。
- 運用上の変更点は「外部通信不要化」のみ。

## 9. 受け入れ条件
1. Mermaid図を含むMarkdownを、インターネット未接続環境でPDF化できる。
2. 変換中にCDN（jsdelivr等）への参照コードが存在しない。
3. package.jsonに `mermaid` がdependenciesとして追加されている。
4. 既存e2e相当シナリオ（Mermaid含む）で変換が成功する。
5. 既存CLIオプションの互換性が維持される。

## 10. テストケース要約
- Unit
  - HTMLテンプレートにCDNスクリプトが含まれないことを検証する。
  - Mermaid初期化スクリプトが有効であることを検証する。
- E2E
  - Mermaidを含むサンプルMarkdownからPDFが生成できること。
  - ネットワーク遮断条件でも変換成功すること。

## 11. ロールアウト計画
1. v1.1.0として仕様確定。
2. 実装でローカル依存読込へ置換。
3. 既存テストに加えてオフライン観点の検証を追加。
4. 品質ゲート通過後にリリースする。
